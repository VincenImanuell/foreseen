CELO sdk pr-template-checklist: `.github/PULL_REQUEST_TEMPLATE.md`'s checklist is conditional per subtree (`forge test` "if contracts changed", `pnpm lint`+`pnpm build` "if frontend changed", `npm test` "if SDK changed") rather than one fixed list — a docs-only or single-subtree PR isn't forced to claim it ran checks for code it never touched, which would otherwise train contributors to rubber-stamp the checklist.
