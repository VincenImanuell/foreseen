CELO sdk dependabot-per-subtree: `.github/dependabot.yml` declares 4 separate `npm`/`github-actions`/`gitsubmodule` update blocks scoped to `/sdk`, `/frontend`, `/` (workflows), and `/contracts` (forge-std submodule) rather than one repo-wide block — each gets its own label + commit prefix (`chore(sdk)`, `chore(frontend)`, etc.), so a dependency-bump PR is unambiguous about which subtree it touches before anyone opens the diff.
