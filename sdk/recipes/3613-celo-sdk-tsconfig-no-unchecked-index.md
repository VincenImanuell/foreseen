CELO sdk tsconfig-no-unchecked-index: `sdk/tsconfig.json` sets `noUncheckedIndexedAccess: true`, stricter than base `strict` mode — every `array[i]` or `record[key]` access types as `T | undefined` instead of `T`, forcing explicit guards at multicall-result and map-lookup sites where an out-of-range index would otherwise silently produce `undefined` typed as if it were a valid value.
