CELO sdk ci-contracts-scope: `ci.yml` has no `paths:` filter (unlike `sdk.yml`/`frontend.yml`), so it fires on every push/PR to `main`/`develop` regardless of which subtree changed — deliberate, since contract correctness (`forge fmt --check`, `forge build --sizes`, `forge test -vvv`) is cheap and worth re-checking even on unrelated SDK/frontend commits to catch accidental contract edits.
